### Introdução

A **estratégia de *tiling***, também conhecida como *blocking*, é uma técnica fundamental para otimizar o acesso à memória em kernels CUDA. Ao particionar os dados da memória global em subconjuntos menores, que podem ser processados na memória compartilhada, a estratégia de *tiling* reduz o número total de acessos à memória global, que é mais lenta. Este capítulo explora em detalhes a estratégia de *tiling* em CUDA, como ela particiona os dados da memória global, como cada *tile* é processado pelas threads de um bloco, como ela reduz o número total de acessos à memória global e como ela aumenta o **Compute to Global Memory Access (CGMA) ratio** em aplicações com alta demanda por acesso à memória.

### Particionamento da Memória Global com Tiling

A estratégia de *tiling* consiste em dividir o conjunto total de dados armazenados na memória global em subconjuntos menores chamados **tiles** ou **blocos**. Cada *tile* representa uma porção dos dados que será processada pelas threads dentro de um bloco. O objetivo do *tiling* é carregar esses *tiles* menores na memória compartilhada, que é mais rápida, para reduzir o número de acessos à memória global e aumentar a localidade de dados.

**Conceito 1: Divisão da Memória Global em Tiles**

A estratégia de *tiling* particiona a memória global em subconjuntos menores, chamados *tiles*, que podem ser carregados na memória compartilhada e processados pelos threads em um bloco.

**Lemma 1:** *A estratégia de tiling divide os dados da memória global em subsets menores chamados tiles, que podem ser carregados na memória compartilhada para processamento local, reduzindo o número de acessos a memória global.*

*Prova:* A definição de *tiling* implica a divisão dos dados em blocos menores, que podem ser processados localmente. $\blacksquare$

**Conceito 2: Tiles e Blocos de Threads**

Cada *tile* é processado pelas threads dentro de um bloco. Os dados de um *tile* são carregados na memória compartilhada e, então, são usados pelas threads que executam o código do kernel naquele bloco. Cada bloco processa seu próprio *tile*, e o processo é repetido para todos os blocos.

**Corolário 1:** *A estratégia de tiling aloca um tile de dados para cada bloco de threads, de forma que os dados de cada tile são processados pelas threads dentro do bloco correspondente.*

*Derivação:* A definição de *tiling* implica que os dados são particionados, e cada partição é processada por um bloco. $\blacksquare$

### Redução do Acesso à Memória Global

A principal vantagem da estratégia de *tiling* é que ela reduz o número total de acessos à memória global. Em vez de cada thread acessar diretamente a memória global para cada dado necessário, os dados são carregados uma única vez na memória compartilhada por todas as threads do bloco, e as threads podem acessar esses dados diretamente da memória compartilhada. Isso reduz o tempo de espera e os recursos gastos com o acesso à memória global.
